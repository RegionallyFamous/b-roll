WP Desktop Mode is the desktop

ODD turns WordPress into an app platform.

WP Desktop Mode gives WordPress a real desktop — windows, a dock, shortcuts. ODD goes further: it's a runtime that lets you package any JavaScript web app into a .wp bundle and install it into WordPress in one click. With animated wallpapers, icon packs, and a control panel with personality layered on top.

A real app runtime for WordPress

Package any web app — Vite, Next export, plain HTML/JS — into a .wp bundle. ODD installs it into WordPress, gives it a dock icon and a desktop window, and runs it in a sandboxed, capability-aware iframe. No PHP. No admin pages. No build pipeline surgery.

An extension API your apps can plug into

Register commands for the ⌘K palette. Drop widgets onto the desktop. Subscribe to the shared event bus (wp.hooks) and store. Ship your own wallpapers, icon sets, and Iris personalities — all through the same manifest format ODD itself uses.

The pretty layer, for free

Animated PixiJS wallpaper scenes, seventeen coordinated icon packs, a control panel with personality, and Iris — a reactive desktop muse — all ride along in the same plugin. The stage is set before your app even opens.

Build your own app

WordPress is suddenly a great place to ship software.

The web already has a browser. WordPress already has auth, users, capabilities, a REST API, and millions of installs. What it didn't have was a way to ship apps on top of all that — apps that feel like software, not admin pages. ODD is that layer.

Write it however you want

Any framework or none. Vite, Next static export, Svelte, Solid, htmx, vanilla. If it builds to HTML + JS + CSS, it runs.

Ship it as a .wp file

A zip with a manifest.json + your bundle. Bazaar-compatible — the same file works in any host that speaks the format.

Install with one click

Drop the file into the Apps panel, or paste a URL. ODD unpacks it, registers the icon, and wires up the dock + desktop shortcut + command-palette entry.

React is already there

Bare import React from 'react' works out of the box — ODD injects an import map and remaps it to WordPress's bundled React. You don't ship a copy, you don't fight module resolution.

You get the user's capabilities

Apps run in a same-origin iframe with cookie auth. Hit the WordPress REST API as the logged-in user, with their capabilities. No extra plumbing.

Host your own catalog

Any URL that returns a registry.json is an ODD app store. Point your customers at it; new releases pull in automatically.

Seven apps, shipped as examples

If the runtime is this capable, what does an app actually look like?

ODD ships with seven working apps in its catalog. They're real — install and use them today — but they're also reference implementations. Every one of them is a recipe for "how do I build an app like this?" Source, manifests, and bundles are all in the repo.

Mosaic

Pixel editor with drawing tools, fills, undo/redo, and PNG export when your 32×32 masterpiece is ready to ship.

creativeeditor

Flow

Pomodoro with ambient soundscapes, per-session task lists, and a heatmap showing exactly which days you crushed it.

productivityfocus

Board

Cards. Columns. Drag. Drop. Done. A no-nonsense project board that shows what's moving, what's stuck, and what's finally done.

productivitykanban

Ledger

Track clients, generate polished invoices, and log payments without ever leaving WordPress. Get paid, inside the admin.

businessinvoicing

Swatch

HSL editing, harmony rules, WCAG contrast checks, and one-click export to CSS, Tailwind, or SVG. Pick colors like you mean it.

designpalette

Sine

A real Web Audio synthesizer: four waveforms, ADSR envelope, resonant filter, and a step sequencer. Yes, it makes noise.

musicsynth

Tome

A fast internal wiki for your team. Write in markdown, organize into pages, find anything instantly. Docs that don't leave WordPress.

wikimarkdown
